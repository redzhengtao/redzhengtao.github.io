spring 自动装配Bean

pojo下三个类，cat,dog,people

package com.kuang.pojo;
public class Cat {
    public void shout() {
        System.out.println("miao~");
    }

}

package com.kuang.pojo;
public class Dog {
    public void shout() {
        System.out.println("wang~");
    }
}

package com.kuang.pojo;
public class People {
    private Cat cat;
    private Dog dog;
    private String name;

beans配置文件

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
        <bean id="cat" class="com.kuang.pojo.Cat"></bean>
        <bean id="dog" class="com.kuang.pojo.Dog"></bean>

        <bean id="people" class="com.kuang.pojo.People">
            <property name="name" value="小狂神"></property>
            <property name="cat" ref="cat"></property>
            <property name="dog" ref="dog"></property>
        </bean>
</beans>

测试类

import com.kuang.pojo.People;
import org.junit.Test;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class t {
    @Test
    public void ttt(){
        ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
        People people = context.getBean("people", People.class);
        people.getDog().shout();
        people.getCat().shout();
    }
}

运行结果

wang~
miao~

Process finished with exit code 0

如果能在bean里自动寻找cat和dog的话，自动装配，就能减少代码量

更改配置文件

    <bean id="cat" class="com.kuang.pojo.Cat"></bean>
    <bean id="dog" class="com.kuang.pojo.Dog"></bean>

    <bean id="people" class="com.kuang.pojo.People" autowire="byName">
        <property name="name" value="小狂神"></property>
    </bean>

测试成功！

wang~
miao~

小结：

当一个bean节点带有 autowire byName的属性时。
将查找其类中所有的set方法名，例如setCat，获得将set去掉并且首字母小写的字符串，即cat。
去spring容器中寻找是否有此字符串名称id的对象。
如果有，就取出注入；如果没有，就报空指针异常。

下面介绍 autowire byType (按类型自动装配)

更改配置文件

    <bean id="cat" class="com.kuang.pojo.Cat"></bean>
    <bean id="dog" class="com.kuang.pojo.Dog"></bean>

    <bean class="com.kuang.pojo.People" autowire="byType">
        <property name="name" value="小狂神"></property>
    </bean>

测试类此时只需要获取对象类型，甚至不用id

    ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
    People people = context.getBean(People.class);
    people.getDog().shout();
    people.getCat().shout();

Javaweb Response下载文件

自己的源码

public class FileServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        String realPath = "D:\\work\\kuang\\servlet\\javaweb\\response\\src\\main\\resources\\小小红.jpg";
        String fileName = realPath.substring(realPath.lastIndexOf("\\") + 1);
        resp.setHeader("Content-Disposition","attachment;filename="+ URLEncoder.encode(fileName,"UTF-8"));
        FileInputStream in = new FileInputStream(realPath);
        int len=0;
        byte[] buffer=new byte[1024];
        ServletOutputStream out = resp.getOutputStream();
        while((len=in.read(buffer))>0){
            out.write(buffer,0,len);
        }
        in.close();
        out.close();

    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        super.doPost(req, resp);
    }
}

Javaweb Servlet原理

Servlet原理

这里扩展一下写404网页

先写ErrorServlet类

    package com.red.servlet;

    import javax.servlet.ServletException;
    import javax.servlet.http.HttpServlet;
    import javax.servlet.http.HttpServletRequest;
    import javax.servlet.http.HttpServletResponse;
    import java.io.IOException;
    import java.io.PrintWriter;

    public class ErrorServlet extends HttpServlet {
        @Override
        protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
            resp.setContentType("text/html");
            resp.setCharacterEncoding("utf-8");

            PrintWriter writer = resp.getWriter();
            writer.print("<h1>404</h1>");
        }

        @Override
        protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
            super.doPost(req, resp);
        }
    }

再编辑web.xml

    <web-app>
      <display-name>Archetype Created Web Application</display-name>
      
      <!--编写Servlet的映射-->
      
      
      <servlet>
        <servlet-name>hello</servlet-name>
        <servlet-class>com.red.servlet.HelloServlet</servlet-class>
      </servlet>
      <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello</url-pattern>
      </servlet-mapping>
         <!--映射到错误类-->
         
         
      <servlet>
        <servlet-name>error</servlet-name>
        <servlet-class>com.red.servlet.ErrorServlet</servlet-class>
      </servlet>
      <servlet-mapping>
        <servlet-name>error</servlet-name>
        <!-- 这里是正则 *代表任意字符-->
        <url-pattern>/*</url-pattern>
      </servlet-mapping>


    </web-app>

运行tomcat服务器，打开网页

修改url路径到hello，还是会找到 HelloServlet 网页

原因是优先级问题：

指定了固有的映射路径优先级最高，如果找不到就会走默认的处理请求

css 属性选择器

默认

                                    .demo a{
                                        float:left;
                                        display: block;
                                        height: 50px;
                                        width: 50px;
                                        border-radius: 10px;
                                        background: #79c7c7;
                                        text-align: center;
                                        color: gainsboro;
                                        text-decoration: none;
                                        margin-right: 5px;
                                        font: bold 20px/50px Arial;
                                    }

常用属性选择器：

                            = 
                            *=
                            ^=
                            $=

存在id属性的元素

                                        a[id]{
                                background: yellow;
                            }

1、el[attr=val] 选择 attr 为 val 的 el 元素

                                    a[href = md]{
                                       background: wheat;
                                   }


                                    或者id的first的元素
                                    a[id=first]{
                                        background: #66945e;
                                    }

2、el[attr *= val] 只要el 元素的 attr 属性值里含有 val 就能被选择

                                    a[class*="links"]{
                                       background: yellow;
                                   }

3、el[attr ~= val] el元素中的 attr 属性的属性值 val 单独存在 或 存在的属性值与其他属性值之间用空格隔开就能被选择

                                    a[href ~=abc]{
                                           background: wheat;
                                       }

4、el[attr ^= val] el 元素的 attr 属性值里含有 val 且 val 在最前面才能被选择

                                    a[href^=http]{
                                            background: #132456;
                                        }

5、el[attr $= val] el 元素的 attr 属性值里含有 val 且 在最后面

                                    a[href$=doc]{
                                           background: wheat;
                                       }

6、el[attr |= val] el 元素中 attr 属性的属性值单独存在或属性值之间用 “-” 隔开且属性值 val 在 “-” 前

                                 a[href |=abc-2]{
                                       background: wheat;
                                   }

最常用的有 = *= ^= $= ,其他比较少用

css 结构伪类选择器

结构伪类选择器主要根据文档结构来选择器元素，常用于根据父级选择器里面的子元素

选择符 解析
E:first-child 匹配父元素中的第一个子元素 E
E:first-child 匹配父元素中最后一个E元素
E:nth-child(n) 匹配父元素中的第n个子元素E
E:first-of-type 指定类型E的第一个
E:last-of-type 指定类型E的最后一个
E:nth-of-type(n) 指定类型E的第n个 
        ul li:first-child{
        background: #66945e;
    }


        ul li:last-child{
        background: #c75a5a;
    }


         p:nth-child(2){
        background: blue;
    }


         p:nth-of-type(1){
        background: #c75a5a;
    }


        同理也有
            ul li:first-of-type{
        background: #e5e39d;
    }
        和
            ul li:last-of-type{
        background: #e5e39d;
    }

css 层次选择器

1、后代选择器

        body p{
        background: #79c7c7;
      }
      在某个元素的后面

2、子选择器

        body >p{
        background: #79c7c7;
      }
      一代 儿子

3、相邻兄弟选择器

        .green + h1{
          background: #66945e;
      }
        指下一个标签

4、通用选择器

         .green ~ h1{
          background: #66945e;
      }
      它的功能是将后续所有的兄弟（相同等级的）标签背景全部改为green色：
